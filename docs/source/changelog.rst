Changelog
=========

ETLPlus publishes release artifacts from Git tags, and the historical release notes are currently
maintained on GitHub Releases.

Current Release
---------------

The documentation published here tracks ETLPlus |release|.

Release History
---------------

- `GitHub Releases <https://github.com/Dagitali/ETLPlus/releases>`_
- `Git tags <https://github.com/Dagitali/ETLPlus/tags>`_

Documentation Publishing
------------------------

The published docs site includes:

- Getting-started pages for installation, quickstart usage, and compatibility expectations
- Guide pages for examples, pipeline authoring, file-handler coverage, and contributor workflows
- An autogenerated API reference sourced from ``etlplus`` docstrings
- CI validation for the HTML, EPUB, and linkcheck builders advertised to Read the Docs

Roadmap: v1.0.0
---------------

Highlights
~~~~~~~~~~

- Establishes the first stable `v1.x` support line for the documented ETLPlus CLI and Python API.
- Publishes a release path that validates distributions, audits packaged artifacts, smoke-tests the
	built wheel, and publishes tagged releases through dedicated automation.
- Documents the supported Python versions, support policy, compatibility expectations, and stable
	surface area for maintainers and users.

Stability Contract
~~~~~~~~~~~~~~~~~~

The `v1.0.0` release support promise covers:

- The documented CLI commands: ``check``, ``extract``, ``validate``, ``transform``, ``load``,
	``render``, and ``run``
- The documented ETL primitives in ``etlplus.ops``
- The implemented file handlers advertised as implemented in the handler matrix
- The documented API client and pagination helpers in ``etlplus.api``

The following remain outside the stable contract unless explicitly promoted in a later release:

- Stubbed or placeholder file handlers and format adapters
- Placeholder database execution paths that are documented as non-final
- Defunct or migration-reference modules retained for historical context

Operational Notes
~~~~~~~~~~~~~~~~~

- Supported Python versions: 3.13 and 3.14
- Release artifacts are built from Git tags via ``setuptools-scm``
- CI validates linting, docstring linting, tests, type-checking, docs builds, artifact audits, and
	clean-environment wheel installs before release publication
